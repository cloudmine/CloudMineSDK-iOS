Detailed Description

Encodes objects and scalar values into a dictionary form. This is meant for simple domain model relationships and should not be considered as general purpose as NSArchiver and NSUnarchiver, which can be used easily to serialize entire user interfaces.

Other than that note, this works the exact same way as standard Cocoa object serialization and deserailization works. Just implement NSCoding on your class and respond to the two methods.

Note: You cannot use this class directly to serialize objects to be sent over the wire. You must use one of its subclasses appropriate for the particular over-the-wire format you would like to use.

Member Function Documentation

+ (NSDictionary *) encodeObjects: (id< NSFastEnumeration >)  objects

Kicks off the encoding process for a collection of objects that implement CMSerializable.

This return type should be the same as for encodedRepresentation. Calling this method kicks off the entire encoding process and returns the final encoded value. Subclasses MUST override this method for reasons outlined in the description for encodedRepresentation.

Property Documentation

- (id) encodedRepresentation [read, assign]

The encoded representation of the object. This will be an NSDictionary in CMObjectEncoder and an encoding-method-specific type for subclasses.

This MUST be overridden in subclasses. When serializing an object it cannot be sent as an instance of an NSDictionary and subclasses of this must take care of that conversion.
