typedef void(^ CMUserFetchCallback)(NSArray *users, NSDictionary *errors)

The block callback for any user account operation that involves fetching one or more user profiles. The block returns void and takes an NSArray containing all the deserialized CMUser (or subclass) instances as well as a dictionary of error messages the server sent back. The second parameter will always be an empty dictionary except when using userWithIdentifier:callback: (CMUser), in which case that will be the place where the "not found" error is recorded.

typedef void(^ CMUserOperationCallback)(CMUserAccountResult resultCode, NSArray *messages)

The block callback for all user account and session operations that take place on an instance of CMUser. The block returns void and takes a CMUserAccountResult code representing the reuslt of the operation, as well as an array of messages the server sent back. These messages will more often than not be errors.

Use the convenience functions CMUserAccountOperationSuccessful and CMUserAccountOperationFailed to help you see if resultCode represents success or failure.
