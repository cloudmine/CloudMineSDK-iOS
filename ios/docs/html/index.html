CloudMine iOS SDK Documentation

Important

This framework uses ARC (Automatic Reference Counting) and is thus compatible only with XCode 4.1 or higher and iOS 4 or higher.

Introduction

The native iOS library allows you to interact with your CloudMine remote data store, create and manage user accounts, and execute your custom server-side code snippets in a clean, object-oriented fashion, without having to manage connection pools or assemble URLs.

Before getting started, you must first use the CMAPICredentials singleton to configure your app's identifier and secret from your CloudMine Dashboard.

The most important classes to know are CMStore and CMUser. CMStore is like a box for all your app's models. All of your storage and retrieval operations are coordinated through it. CMUser represents an account for a user of your app. See the framework reference for more details on users, security, and session tokens.

Each of your model classes that you want to be managed by a CMStore must either extend CMObject or implement CMSerializable. The former provides an easier-to-use turnkey implementation, and is recommended over implementing the latter yourself. You must also implement initWithCoder: and encodeWithCoder: in all CMObject subclasses so the framework knows how to serialize your objects.

If you need to store geospatial information within any of your objects, use an instance of the provided CMGeoPoint class.

All operations that involve a remote call are performed asynchronously. This means that you don't need to worry about blocking the UI thread when interacting with CloudMine. Each one of these methods takes one or more callback blocks as the last parameter(s) that will be executed when the operation completes.

The framework has the following external framework dependencies, which you must setup in your app's XCode project:

  • CFNetwork
  • SystemConfiguration
  • MobileCoreServices
  • CoreGraphics
  • UIKit
  • libz
