Detailed Description

Extend from this instead of NSObject for all model objects in your app that need to be backed by remote storage on CloudMine. Be sure to implement initWithCoder: and encodeWithCoder: in each of these model classes to define how to serialize and deserialize the object. This behavior follows the standard archiving/unarchiving conventions defined by Apple's NSCoding protocol.

This will also take care of generating a default objectId for you, in the form of a UUID.

- (void) save: (CMStoreObjectUploadCallback)  callback

Saves this object to CloudMine using its current store. If this object does not belong to a store, the default store will be used. It will be added at the app-level. If you need to associate this object with a user, see saveWithUser:callback:.

If this object already belongs to a store, it will be saved to the app- or user-level, whichever it was added as. For example, if it was originally added to a store using addObject: (CMStore) or CMStore::saveObject:: (i.e. at the app-level) it will be saved at the app-level. If it was originally added using CMStore::addUserObject:callback:, saveUserObject:callback: (CMStore), or saveWithUser:callback: (i.e. at the user-level) it will be saved at the user-level.

Parameters
callbackThe callback block to be invoked after the save operation has completed.
