Detailed Description

Extend from this instead of NSObject for all model objects in your app that need to be backed by remote storage on CloudMine. Be sure to implement initWithCoder: and encodeWithCoder: in each of these model classes to define how to serialize and deserialize the object. This behavior follows the standard archiving/unarchiving conventions defined by Apple's NSCoding protocol.

This will also take care of generating a default objectId for you, in the form of a UUID.

- (CMStore*) store [read, write, retain]

The store that the object belongs to. It is possible for this to be nil, meaning that the object was created locally and hasn't been saved via a store yet. It is important to note that the object must belong to a store if you need to save it. You can easily add the object to a store and save it by using CMStore's saveObject: method.

If you manually change the store yourself, this object will automatically remove itself from the old store and add it to the new store. This operation is thread-safe.
