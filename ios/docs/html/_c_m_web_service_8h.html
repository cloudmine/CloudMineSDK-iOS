typedef void(^ CMWebServiceFetchFailureCallback)(NSError *error)

Callback block signature for all operations on CMWebService that can fail. These are general errors that cause the entire call to fail, not key-specific errors (which are reported instead in the errors dictionary in the CMWebServiceObjectFetchSuccessCallback callback block). The block returns void and takes an NSError describing the error as an argument.

typedef void(^ CMWebServiceFileFetchSuccessCallback)(NSData *data, NSString *contentType, NSDictionary *headers)

Callback block signature for all operations on CMWebService that download binary files from the CloudMine servers. These blocks return void and take an NSData instance that contains the raw data for the file as well as a string with the content type of the file returned from the server.

typedef void(^ CMWebServiceObjectFetchSuccessCallback)(NSDictionary *results, NSDictionary *errors, NSDictionary *meta, id snippetResult, NSNumber *count, NSDictionary *headers)

Callback block signature for all operations on CMWebService that fetch objects from the CloudMine servers. These blocks return void and take a dictionary of results, a dictionary of errors, a dictionary of metadata, and a dynamic snippet result as arguments. These map directly with the CloudMine API response format.

typedef void(^ CMWebServiceUserAccountOperationCallback)(CMUserAccountResult result, NSDictionary *responseBody)

Callback block signature for all operations on CMWebService and CMUser that involve the management of user accounts and user sessions. These blocks return void and take a CMUserAccountResult code that represents the result of the operation as well as an NSDictionary containing the dictionary representation of the response body from the server.
